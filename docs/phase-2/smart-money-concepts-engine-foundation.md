# Phase 2.6 — Smart Money Concepts Engine Foundation

## Objective

Add an independent candle-structure directional voter for:

- confirmed swing highs and lows;
- Break of Structure (BOS);
- Change of Character (CHoCH);
- buy-side and sell-side liquidity sweeps;
- order-block zones;
- fair-value-gap zones.

The engine consumes canonical, closed five-minute candles and produces immutable point-in-time evidence for Thesis/Fusion.

It cannot create a canonical signal, approve risk, construct a trade plan, mutate a portfolio, submit an order, or contact Upstox directly.

## Methodology boundary

Smart Money Concepts terminology is not exchange-standardized. Different charting systems can produce different BOS, CHoCH, order-block and fair-value-gap labels from the same candles.

This implementation is therefore explicitly a **deterministic heuristic policy**:

```text
engine_code: THESIS_PULSE_SMART_MONEY_CONCEPTS
engine_version: 1.0.0
policy_version: smart-money-structure-v1.0.0
methodology: CONFIRMED_PIVOT_STRUCTURE_HEURISTIC
```

Every output includes:

```text
SMC_RULESET_IS_DETERMINISTIC_HEURISTIC
```

The label does not claim that an institution placed, accumulated or distributed an order. It describes price structure generated by the documented policy.

A future policy may use another pivot width, displacement rule or zone definition. It must use a new policy version and must not rewrite historical V1 outputs.

## Authority

The engine registry records:

```text
engine_role: DIRECTIONAL_VOTER
owner_service: ThesisPulse.AI
can_create_signals: false
can_execute_orders: false
```

Seed `S0013` registers the authority. Verification `S0009` rejects authority drift.

## Processing path

```text
canonical market.candle.published.v1
  -> Feature Factory intake
  -> Smart Money point-in-time candle window
  -> confirmed pivots
  -> BOS / CHoCH / sweep detection
  -> order-block and fair-value-gap zones
  -> immutable intelligence.engine_outputs
  -> exact intelligence.engine_output_market_inputs lineage
  -> optional SMART_MONEY_CONCEPTS Fusion vote
```

SQL Server remains the operational source of truth.

## Point-in-time boundary

For source candle cutoff `T`, the SQL store selects only candles where:

```text
candle.close_at_utc <= T
candle.received_at_utc <= source publication occurred_at_utc
candle.is_closed = true
```

For each candle open time, the highest revision available at that historical cutoff is selected.

The engine therefore cannot consume:

- a candle that closed after the source candle;
- a correction that arrived after the source event;
- a provisional candle;
- a future-confirmed pivot.

Every selected candle ID is written to `intelligence.engine_output_market_inputs`. The trigger candle is `PRIMARY`; prior candles are `CONTEXT`.

## Confirmed swing policy

Default pivot width:

```text
left bars: 2
right bars: 2
```

A swing high at index `i` requires:

```text
high[i] > every high in the two candles before i
high[i] > every high in the two candles after i
```

A swing low uses the equivalent strict-low rule.

The pivot becomes usable only after the final right-side candle closes. Equal highs or equal lows do not create a pivot in V1.

## Structure state

The most recent two confirmed highs and lows define the state:

| Highs | Lows | State |
|---|---|---|
| Higher high | Higher low | `BULLISH` |
| Lower high | Lower low | `BEARISH` |
| Mixed | Mixed | `RANGE` |
| Insufficient confirmed pivots | Insufficient | `UNCONFIRMED` |

## BOS and CHoCH

A break requires a candle **close** beyond the latest confirmed swing level plus the configured tolerance.

Default tolerance:

```text
0.0002 = 0.02%
```

Bullish close through a swing high:

- `CHOCH` when prior structure was `BEARISH`;
- otherwise `BOS`.

Bearish close through a swing low:

- `CHOCH` when prior structure was `BULLISH`;
- otherwise `BOS`.

The same confirmed swing cannot repeatedly create new break events.

## Liquidity sweeps

A wick violation without a confirming close is classified as a sweep, not a structure break.

### Buy-side sweep

```text
high > confirmed swing high + tolerance
close <= confirmed swing high
directional implication: SHORT
```

### Sell-side sweep

```text
low < confirmed swing low - tolerance
close >= confirmed swing low
directional implication: LONG
```

The same swing cannot repeatedly create new sweep events.

## Order blocks

An order block is created only after a BOS or CHoCH.

### Bullish order block

The engine searches backward from a bullish structure event for the latest bearish candle within the configured search window.

```text
zone lower = candle low
zone upper = candle open
```

### Bearish order block

The engine searches backward from a bearish structure event for the latest bullish candle.

```text
zone lower = candle open
zone upper = candle high
```

Default search window:

```text
8 candles
```

Zone states:

- `ACTIVE`: price has not revisited the zone;
- `MITIGATED`: price has entered the zone but has not closed beyond invalidation;
- `INVALIDATED`: a candle closes through the far boundary.

## Fair-value gaps

V1 uses a three-candle imbalance definition.

### Bullish FVG

```text
current low > high from two candles earlier
zone = earlier high to current low
```

### Bearish FVG

```text
current high < low from two candles earlier
zone = current high to earlier low
```

The gap size must satisfy the configured minimum fraction.

Default:

```text
0.0005 = 0.05%
```

A complete traversal of the gap invalidates it; partial entry marks it mitigated.

## Scoring

Initial component weights:

| Component | Weight |
|---|---:|
| Confirmed structure state | 0.20 |
| Recent BOS or CHoCH | 0.35 |
| Recent liquidity sweep | 0.20 |
| Active or mitigated order block | 0.15 |
| Active or mitigated fair-value gap | 0.10 |

CHoCH receives the full structure-event weight. BOS receives 80% of it. Mitigated zones receive 50% of their zone weight.

The final score is bounded to `[-1, 1]`.

```text
score >=  0.20 -> LONG
score <= -0.20 -> SHORT
otherwise       -> NEUTRAL
```

Confidence combines:

- candle-window completeness;
- valid-input ratio;
- confirmed-pivot coverage;
- score magnitude;
- component agreement;
- event recency.

`RANGE` and `UNCONFIRMED` structure reduce confidence.

## Fusion eligibility

An SMC output can enter Fusion only when:

- the source is a closed five-minute candle;
- the exact source revision exists in the point-in-time window;
- the required candle count is present;
- the valid-input ratio passes policy;
- no intraday candle gap is detected;
- the output is fresh;
- both a confirmed swing high and swing low exist;
- direction is not neutral;
- confidence passes the Fusion threshold;
- a recent BOS, CHoCH or liquidity sweep exists.

Default gates:

```text
required candles: 30
maximum candle window: 128
minimum valid-input ratio: 0.95
maximum output age: 420 seconds
minimum Fusion confidence: 0.55
```

Missing or ineligible SMC evidence contributes no vote. It is never converted to neutral confirmation or agreement.

## Idempotency and revisions

The same source candle publication UID returns the original output.

A corrected candle publication with a new message UID and a higher candle revision:

- recalculates the historical point-in-time window;
- creates the next output revision;
- marks the former output non-current;
- stores `supersedes_engine_output_uid`;
- retains both outputs for audit and replay.

## Rollback

Disable the engine without stopping candle ingestion or the other intelligence engines:

```text
THESISPULSE_SMART_MONEY_ENGINE_ENABLED=false
```

Historical outputs remain immutable.

## Exit gate

- Confirmed pivots never use future-unconfirmed structure.
- Wick sweeps cannot become BOS without a close confirmation.
- Bullish and bearish BOS are deterministic.
- Bullish-to-bearish and bearish-to-bullish CHoCH are deterministic.
- Order blocks and fair-value gaps have auditable zone boundaries and status.
- Insufficient, stale, incomplete and mismatched inputs fail closed.
- Duplicate source messages are idempotent.
- Corrected candles create revisioned outputs.
- Exact candle lineage is persisted.
- Only eligible SMC output enters Fusion.
- Engine authority verification passes.
- Python, .NET, database and React CI are green.
